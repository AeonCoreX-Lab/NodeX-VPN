NodeX VPN
CLI Reference

Full reference for the nodex command-line client. Covers installation, all commands, bridge mode, router setup, and scripting.

Installation

via npm (recommended)

The easiest way. npm selects the correct platform binary automatically.

shell
# Global install npm install -g @aeoncorex/nodex-vpn # Run without installing npx @aeoncorex/nodex-vpn version

Manual Download

Download the binary for your platform from GitHub Releases.

Linux / Termux
tar -xzf nodex-linux-x86_64-v0.1.0.tar.gz sudo mv nodex-linux-x86_64-v0.1.0/nodex /usr/local/bin/ nodex version # Termux (no sudo) mv nodex ~/.local/bin/
macOS
tar -xzf nodex-macos-arm64-v0.1.0.tar.gz sudo mv nodex-macos-arm64-v0.1.0/nodex /usr/local/bin/ nodex version
Windows (PowerShell)
Expand-Archive nodex-windows-x86_64-v0.1.0.zip # Add the extracted folder to your PATH, then: nodex.exe version

Quick Start

shell
# 1. Log in with Google $ nodex login ✔ Logged in as user@gmail.com # 2. Connect (auto exit country) $ nodex connect [██████████████████████████████] 100% Connected! # 3. Connect with specific country $ nodex connect --country DE # 4. Check status $ nodex status # 5. Disconnect (Ctrl+C while connected, or) $ nodex disconnect

Authentication

NodeX VPN uses Google OAuth2 PKCE for authentication. When you run nodex login, a browser window opens and you sign in with your Google account. The token is stored encrypted in your OS keychain — never sent to any NodeX server.

Note: Authentication is only required for the CLI. Mobile and desktop apps have their own sign-in flow.

nodex login

Authenticate with your Google account. Opens a browser for OAuth2 sign-in. Stores the token securely in the OS keychain.

shell
$ nodex login › Opening browser for Google Sign-In… › Waiting on localhost:9005… ✔ Logged in as user@gmail.com Name: Your Name

If the browser doesn't open automatically (e.g. headless Termux), the URL is printed to the terminal for manual opening.

nodex logout

Remove saved credentials from the OS keychain.

shell
$ nodex logout ✔ Logged out successfully.

nodex whoami

Show the currently authenticated account and token validity.

shell
$ nodex whoami Email: user@gmail.com Name: Your Name ✔ Token valid

nodex connect

Connect to NodeX VPN. Routes all TCP traffic through a Tor circuit. Shows real-time bootstrap progress and live stats while connected.

Usage
nodex connect [OPTIONS] -c, --country <ISO> Exit country (US, DE, NL, JP, GB…) -b, --bridge <LINE> obfs4 bridge line (repeatable) --socks <ADDR> SOCKS5 listen addr [default: 127.0.0.1:9050] --dns <ADDR> DNS-over-Tor addr [default: 127.0.0.1:5353] --state-dir <DIR> Tor state dir [default: ~/.nodex/state] --cache-dir <DIR> Tor cache dir [default: ~/.nodex/cache] -v, --verbose Debug logging --quiet No banner, pipe-friendly
OptionDefaultDescription
--country ISOautomaticISO 3166-1 alpha-2 exit country code
--bridge LINEnoneobfs4 bridge line. Can be repeated for multiple bridges.
--socks ADDR127.0.0.1:9050SOCKS5 proxy listen address and port
--dns ADDR127.0.0.1:5353DNS-over-Tor listen address and port
--state-dir~/.nodex/stateTor state directory (guard keys, etc.)
--cache-dir~/.nodex/cacheTor cache directory (consensus, descriptors)
--verbose / -voffEnable debug-level log output
--quietoffSuppress banner. Respects NO_COLOR.
Examples
# Auto exit country nodex connect # Germany exit node nodex connect --country DE # Bridge mode (for censored networks) nodex connect --bridge "obfs4 1.2.3.4:443 FINGERPRINT cert=... iat-mode=0" # Listen on all interfaces (router/proxy mode) nodex connect --socks 0.0.0.0:9050 --dns 0.0.0.0:5353 # Quiet mode for scripting nodex connect --country NL --quiet

nodex status

Show current connection status, exit IP, throughput, and circuit info.

Output
$ nodex status Status : ✔ Connected Bootstrap : 100% Done Exit country: DE Exit IP : 185.220.x.x Uptime : 12m 34s ↑ Upload : 2.14 MB/s ↓ Download : 4.87 MB/s Sent total : 94.3 MB Recv total : 312.1 MB Latency : 198 ms Circuits : 3 active 1 pending

nodex nodes

List available VPN exit nodes with latency and load. Load is color-coded: green ≤40%, yellow 41–70%, red >70%.

Usage
nodex nodes [--country <ISO>] [--bridges] # Examples nodex nodes nodex nodes --country NL nodex nodes --bridges

nodex logs

Show recent engine log output. Useful for debugging bootstrap or connection issues.

Usage
nodex logs [--lines <N>] # Show last 50 lines (default) nodex logs # Show last 200 lines nodex logs --lines 200

nodex version

Show full version, build target, and build date.

Output
$ nodex version CLI Version : 0.1.0 Build target: x86_64-unknown-linux-gnu Build date : 2026-04-29 Vendor : AeonCoreX License : MIT

Bridge Mode

If Tor is blocked in your network (China, Iran, Russia, etc.), use bridges with --bridge. Bridges are unlisted Tor relays that disguise your traffic as random bytes using the obfs4 protocol.

Getting Bridge Lines

Get official bridges from bridges.torproject.org — select "obfs4" as the transport type.

Bridge Mode Usage
# Single bridge nodex connect \ --bridge "obfs4 1.2.3.4:443 FINGERPRINT cert=XXXX iat-mode=0" # Multiple bridges (repeat --bridge) nodex connect \ --bridge "obfs4 1.2.3.4:443 FP1 cert=X iat-mode=0" \ --bridge "obfs4 5.6.7.8:9001 FP2 cert=Y iat-mode=0"

Router Setup

Run NodeX VPN on a router to protect all devices on your network automatically — Smart TVs, game consoles, IoT devices — without installing anything on each device.

pfSense / OPNsense (FreeBSD)

shell
# Download FreeBSD binary fetch https://github.com/AeonCoreX/NodeX-VPN/releases/latest/download/nodex-freebsd-x86_64-v0.1.0.tar.gz tar -xzf nodex-freebsd-x86_64-v0.1.0.tar.gz cp nodex-freebsd-x86_64-v0.1.0/nodex /usr/local/bin/ # Listen on all interfaces for LAN routing nodex connect --socks 0.0.0.0:9050 --dns 0.0.0.0:5353

OpenWrt

shell
# Download arm64 binary (for most modern routers) wget https://github.com/AeonCoreX/NodeX-VPN/releases/latest/download/nodex-linux-aarch64-v0.1.0.tar.gz tar -xzf nodex-linux-aarch64-v0.1.0.tar.gz mv nodex /usr/bin/ nodex connect --socks 0.0.0.0:9050
Note: Point your LAN DNS to the router's IP on port 5353 to enable DNS-over-Tor for all devices. Set SOCKS5 proxy on individual apps or configure transparent proxying via iptables.

Scripting & NO_COLOR

NodeX VPN auto-detects terminal color support and respects the NO_COLOR standard.

Color control
# Disable color (piping also disables automatically) NO_COLOR=1 nodex status # Suppress banner for scripting nodex connect --quiet --country DE # Parse status output nodex status --quiet | grep "Exit IP"

Environment Variables

VariableDescription
NO_COLORSet to any value to disable ANSI color output
NODEX_GOOGLE_CLIENT_IDOverride built-in Google OAuth2 Client ID (dev/test)
NODEX_GOOGLE_CLIENT_SECRETOverride built-in Google OAuth2 Client Secret

File Locations

FilePlatformContents
~/.nodex/credentials.jsonLinux / macOS / TermuxOAuth2 token (fallback if OS keychain unavailable)
~/.nodex/state/AllTor state (guard keys, etc.)
~/.nodex/cache/AllTor cache (consensus, descriptors)
%APPDATA%\NodeX\WindowsState and credentials

The credentials file is created with chmod 600 (Unix) — readable only by the current user. On macOS, Windows, and Linux with a secret service, credentials are stored in the OS keychain instead.

Kill Switch

Blocks ALL network traffic if the VPN connection drops — preventing your real IP from being exposed. Uses the OS firewall on each platform.

Requires root / admin. On Linux: sudo nodex connect --kill-switch. On macOS: pfctl requires root. On Windows: run as Administrator.
Usage
# Enable kill switch nodex connect --kill-switch # Kill switch with specific country nodex connect --kill-switch --country DE # Disable is automatic on Ctrl+C / nodex disconnect
PlatformMethod
Linuxnftables (fallback: iptables) — dedicated nodex_kill_switch table
macOSpfctl anchor nodex_kill_switch
Windowsnetsh advfirewall — block-all + allow SOCKS5/Tor rules
FreeBSDpfctl anchor

IPv6 Leak Protection

Disables IPv6 system-wide when the VPN is active. Without this, IPv6 traffic can bypass Tor and reveal your real IP to IPv6-enabled sites.

CLI
nodex connect --kill-switch # kill switch implies IPv6 block on Linux

In mobile/desktop apps: enable "IPv6 Leak Protection" toggle in Settings → Security.

Leak Test

Built-in leak test that checks DNS, IPv6, and exit IP — all routed through Tor for accuracy.

CLI
# Run leak test (must be connected) nodex leak-test # Example output ✔ Exit IP : 185.220.x.x ✔ Exit country : Germany ✔ DNS leak : None ✔ IPv6 leak : None ✔ No leaks detected.

Split Tunneling

Route only specific apps or domains through Tor, while others go direct. Two modes:

ModeBehaviourUse case
BypassListed apps/domains go direct, everything else through TorExclude banking/Netflix
ExclusiveOnly listed apps/domains go through TorTor only for browser
CLI
# Bypass specific domain (go direct) nodex connect --bypass-domain netflix.com --bypass-domain mybank.com # Always bypass LAN (default) # 192.168.0.0/16, 10.0.0.0/8 bypass by default

WebRTC Protection

WebRTC can leak your local/public IP in browsers even when a VPN is active. NodeX detects the risk and provides mitigation steps.

Mobile apps are not affected — system network stack routes all traffic through VPN. The risk is desktop browsers.

Mitigation: Install uBlock Origin → Settings → "Prevent WebRTC from leaking local IP addresses". Or use --kill-switch to block all non-Tor traffic at firewall level.

Node Speed Test

Test latency to exit nodes before connecting. Uses 3 HTTP probes and takes the median. Results cached for the session.

CLI
# Test all nodes and show sorted by latency nodex speed-test # Example output de-fra-01 DE Germany 98 ms Excellent nl-ams-01 NL Netherlands 142 ms Good us-nyc-01 US USA 289 ms Fair

Bandwidth Limiter

Cap upload and/or download rate to prevent the VPN from using all available bandwidth in the background.

CLI
# Limit to 10 Mbps download, 5 Mbps upload nodex connect --max-download 10 --max-upload 5 # No limit (default) nodex connect

Favorite Servers

Bookmark frequently used nodes with a custom label. Sorted by most recently used.

CLI
nodex favorites # list all nodex favorites add de-fra-01 "Work" # add with label nodex favorites remove de-fra-01 # remove nodex connect --favorite Work # connect to favorite

Build from Source

Requires Rust stable (1.70+) and Cargo.

shell
git clone https://github.com/AeonCoreX/NodeX-VPN.git cd NodeX-VPN/rust-core # Set your Google OAuth2 credentials export NODEX_GOOGLE_CLIENT_ID="your-client-id" export NODEX_GOOGLE_CLIENT_SECRET="your-client-secret" # Build CLI cargo build --release --features cli --bin nodex # Binary location ./target/release/nodex version
